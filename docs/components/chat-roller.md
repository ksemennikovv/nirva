# Компонент: Chat Roller

## Что это

Переиспользуемый чат-компонент. Используется на всех страницах с ИИ-диалогом: анализ, рефлексия, дневник. Один компонент — три разных режима работы, управляемых через параметр `mode`.

---

## Ключевые файлы

```
features/chat-roller/
├── index.php (или подключаемый фрагмент)  # HTML-разметка чата
├── chat-roller.css                         # Стили компонента
├── chat-roller.js                          # Логика: отправка, polling, анимации
└── api/
    └── send-message.php                    # Бэкенд — обрабатывает сообщение
```

---

## Режимы работы (mode)

| Режим | Где используется | ИИ-провайдер |
|-------|-----------------|-------------|
| `analysis` | Анализ переживаний | ai_provider_analysis |
| `reflection` | Рефлексия с практикой | ai_provider_analysis |
| `diary` | Дневник | ai_provider_diary |

Режим передаётся при встраивании компонента и отправляется с каждым сообщением в API.

---

## Как встраивать на страницу

```php
$chatMode = 'analysis';
$sessionId = $analysisSession['id'];
require_once 'features/chat-roller/index.php';
```

Компонент читает `$chatMode` и `$sessionId` из контекста.

## Что происходит при отправке сообщения

**Фронтенд (chat-roller.js):**
1. Пользователь вводит текст и нажимает Enter/кнопку
2. Сообщение добавляется в UI мгновенно
3. POST запрос на `features/chat-roller/api/send-message.php`
4. Показывается анимация "думает..."

**Бэкенд (send-message.php):**
1. Сохраняет сообщение пользователя в `messages`
2. Загружает историю диалога
3. Вызывает `AIService::sendMessage()` или `sendWithPrompt()` в зависимости от режима
4. `AIService::checkSafety()` — проверяет кризисность
5. Сохраняет ответ ИИ:
   - Если supervisor mode ON → `review_status = 'pending_review'`
   - Если supervisor mode OFF → `review_status = 'approved'`
6. Возвращает JSON: `{status, message_id}` или `{status: 'pending'}`

**Фронтенд после ответа:**
- Если `status = 'approved'` → показывает ответ ИИ
- Если `status = 'pending'` → запускает polling
- Polling каждые 2 секунды: `GET api/check-message.php?id=...`
- Как только статус `approved` → показывает ответ, останавливает polling

---

## Supervisor Mode — что видит пользователь

Пока ответ ждёт модерации:
- Анимация "думаю..." не пропадает
- Пользователь не может отправить новое сообщение
- Нет таймаута — ждёт сколько нужно

Это намеренно: важно чтобы каждый ответ прошёл проверку на раннем этапе.

---

## Отображение сообщений

- Сообщения пользователя — справа, другой цвет
- Сообщения ИИ — слева
- Markdown поддерживается (жирный, курсив, списки)
- Длинные тексты не обрезаются

---

## Интеграция с профилем и состоянием диалога

`send-message.php` после получения ответа ИИ:
- Обновляет `dialogue_stage` в `analysis_sessions` если нужен переход
- Вызывает `ProfileService::updateProfile()` — ИИ мог предложить обновление профиля
- Обновляет `dialogue_summary` — сжимает длинный диалог для экономии токенов
