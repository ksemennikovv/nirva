# Фича: Дневник (/diary/)

## Что это и зачем

Дневник — лёгкий формат ИИ-диалога между сессиями анализа. Пользователь ведёт его в промежутке: фиксирует наблюдения, события, эмоции. ИИ слушает, задаёт поддерживающие вопросы, помогает осмыслить. Это не глубинная работа как анализ — скорее рефлексивный журнал с живым собеседником.

Дневник влияет на психологический профиль: после разговора ИИ может обновить параметры профиля.

## Как работает

### Создание записи
При открытии `/diary/` создаётся новая запись в `diary_entries`. Каждый раз новая — история хранится как список записей. Возможно ограничение числа бесплатных записей (`DIARY_FREE_ENTRIES_LIMIT` = 10).

### Диалог
Аналогично анализу, через `send-message.php` → `sendDiaryMessage()`:
1. Загружает историю этой записи (`MessageRepository::getMessages($entryId, 'diary')`)
2. Вызывает `AIService::sendWithPrompt()` — с системным промптом для дневника
3. Использует `ai_provider_diary` / `ai_model_diary` из app_settings
4. Сохраняет ответ (с учётом supervisor mode)

### Завершение записи
При завершении:
- `DiaryRepository::updateSummary()` — ИИ формирует краткое резюме разговора
- `ProfileService::updateProfile()` — обновляет профиль если нашёл что-то значимое
- `diary_entries.status = 'completed'`

## Отличие от анализа

| | Анализ | Дневник |
|--|--------|---------|
| Глубина | Глубокая психологическая работа | Лёгкая рефлексия |
| Длительность | Несколько этапов, часто несколько сессий | Один разговор |
| Доступность | Ограничен тарифом (слоты) | Чаще или неограниченно |
| Результат | Анализ + медитация + обновление профиля | Резюме + обновление профиля |
| ИИ-провайдер | ai_provider_analysis | ai_provider_diary |

---

## Отображение на дашборде

На `/dashboard/` показывается блок дневника если:
- Прошло >= `DASHBOARD_DIARY_DAYS_THRESHOLD` (5) дней после последнего анализа
- Или выполнено другое условие из бизнес-логики

Лимит показов блока: `DASHBOARD_DIARY_DAILY_SHOW_LIMIT` = 1 раз в день.

---

## Хранение данных

Сообщения дневника хранятся в той же таблице `messages`, что и сообщения анализа, но с `phase='diary'` и `session_id` = ID записи дневника. Разделение через параметр phase позволяет переиспользовать MessageRepository для всех типов диалогов.
