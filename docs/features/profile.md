# Фича: Психологический профиль

## Что это

Живой психологический портрет пользователя, который автоматически строится и обновляется ИИ в процессе анализов и дневниковых записей. Профиль используется для персонализации: ИИ-диалог учитывает профиль при каждом сообщении, медитации генерируются с учётом профиля.

Пользователь не редактирует профиль вручную — его ведёт ИИ на основе разговоров.

---

## Ключевые файлы

| Файл | Роль |
|------|------|
| `src/services/Profile/ProfileService.php` | Обновление профиля, форматирование для ИИ |

| `database/migrations/` | DDL для profile_parameters, profile_options, user_profile_values, profile_history, profile_memories |

## Структура профиля

### Параметры (profile_parameters)
Справочник того, что можно профилировать. Два типа:
- **fixed** — выбор из предопределённых вариантов (profile_options). Например: "стиль мышления" → [аналитический, интуитивный, эмоциональный...]
- **free** — свободный текст. Например: "жизненные ценности"

Параметры сгруппированы по категориям: личность, отношения, эмоции, цели и т.д.

### Значения пользователя (user_profile_values)
Текущие значения профиля: `userId + parameterId → value` (JSON-массив, может быть несколько значений для одного параметра).

Дополнительно хранит:
- `confidence` (0-1) — уверенность ИИ в этом значении
- `evidence_count` — сколько раз ИИ наблюдал это

### Память (profile_memories)
Свободные наблюдения ИИ, не вписывающиеся в параметры. Например: "пользователь боится конфликтов в семье". Имеют `importance` (0-1) — используется для приоритизации.

### История (profile_history)
Лог всех изменений: что изменилось, откуда (analysis/diary), когда.

## Как профиль обновляется

После каждого завершённого ответа ИИ в анализе или дневнике:

1. `ProfileService::updateProfile($userId, $updates, 'analysis', $sessionId)`:
   - ИИ возвращает предлагаемые обновления в структурированном формате
   - Для fixed-параметров — проверяет что значение есть в `profile_options`
   - Обновляет значения в `user_profile_values`
   - Пересчитывает `confidence` и `evidence_count`
   - Логирует изменение в `profile_history`

2. `ProfileService::addMemories($userId, $memories)`:
   - Сохраняет новые свободные наблюдения в `profile_memories`
   - Старые с низким importance могут вытесняться

## Как профиль используется в диалоге

Перед каждым запросом к ИИ:
- `ProfileService::formatForPrompt($userId)` строит текстовое резюме профиля
- Включает топ-параметры (по evidence_count или последним), отфильтрованные по confidence >= 0.5
- Включает топ-наблюдения (memories) по importance
- Текст вставляется в системный промпт как контекст о пользователе

ИИ использует этот контекст чтобы:
- Не спрашивать то, что уже известно
- Делать более точные и персонализированные наблюдения
- Предлагать практики подходящие конкретному человеку
- Генерировать медитации с правильным тоном и темами
